Meeting the Project Requirements

  1. git respository and publish

    Our project uses the given repository from Dr. Stewart and each of use has commited to the project and each of us has it published in our w3.stu.jmu.edu server

  2. Responsive layouts with media queries:

    We have implemented distinct layouts for large and small screens using CSS media queries in src/css/styles.css. There are comments through out to note where its completed.

  3. Five separate pages minimum:

    Our site includes more than 5 separate pages. Each page has a distinct purpose and content, and they link to each other via the header navigation.

  4. Header:

    All pages include a consistent <header> element containing the project title and navigation links.

  5. Navigation links:

    The <nav> section in the header provides links to all main pages and each page and be changed between through the navigation links

  6. Images with alt attributes:

    We have placed images in the images/ folder and provided descriptive QuickBytes Logo text for each, ensuring accessibility and SEO friendliness.

  7. Validation:

    All HTML files pass the Nu HTML Checker with zero errors or warnings. CSS files have been validated using WAVE Accessibility Tool.

  8. Two different font faces:

    We use 2 different fonts Lucida Console and Times New Roman for.

  9. Form inclusion:

    The tabs.html page contains an order form allowing customers to select items, customize ingredients, and submit. Although not yet functional, it demonstrates proper alignment and use of form elements.

  10. External links:

    Links to external sites are within links.html that link to other food industry related sites.

  11. Network requests (fetch):

    Our network request will call the spotify api so that the user can listen to music while they chose what they would like to eat.
